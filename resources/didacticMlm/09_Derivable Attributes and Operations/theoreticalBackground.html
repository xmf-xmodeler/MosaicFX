2. Theoretical Background:

In the examples and exercises of the preceding learning units, slot values had to be entered manually. In some cases, however, they might also be determined by calculation. The attributes of such slots are called derivable attributes. Derivable attributes are attributes the value of which is either (i) already indirectly available within the model or (ii) dependent on some other attribute with a constant value not yet depicted within the model. That is, the slots of derivable attributes can be calculated rather then entered manually. This calculation requires the use of operations. Operations consist of a signature and an operation body. The signature determines the name of the operation, its input parameters (each of which has a name and a type), and its return type. Operations are specified at the class level (they are a property of classes) and executed at the object level. UML++ uses the Executable Object Constraint Language (XOCL) for the specification of operations. The XOCL is a variant of the widely distributed OCL standard.

Illustration in Tool:

For (i): “already available in model”

  1. Show class and object diagram with derivable attribute of kind (i).
    • For example: number of products ordered, total amount of order
  2. Change object diagram so that derivable slot violates model integrity
  3. Add operation that automatically calculates value

For (ii): “dependent on some other attribute not yet depicted”

  1. Show class and object diagram with derivable attribute of kind (ii) (e.g., age)
  2. Show explanation why this may violate model integrity
  3. Add “base” attribute the derivable attribute is dependent on (e.g., dateOfBirth)
  4. Add operation that automatically calculates derivable slot
  5. Change base attribute to showcase dynamic calculation
