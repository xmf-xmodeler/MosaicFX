7. Adding Rating Attribute and Enumeration

The customer Josie Dough has bought a ticket for the showing of Joker on 7 Nov 2024. But Joker is rated R and may not be watched from anyone under the age of 17. To automatically verify a customer is old enough to watch a particular movie, we must capture a customer’s age and the movie’s rating. Josie’s age is calculated automatically based on her "dateOfBirth".

We must still capture the rating of movie objects, though. First, we add an attribute rating: String to our class movie.

Note, however, that now any string is valid for this attribute. A rating system is restricted to a particular set of values, though. To restrict our attribute to this set of values, we add an enumeration.

Create an enumeration with the name “RatingEnum”. It is good practice to end the name of each enumeration with Enum, so it will not be confused with other data types.

We add three values to our RatingEnum: “G” for general audiences, “PG_13” for parental guidance, and “R” for restricted.


How To Add An Enumeration

To add an enumeration, follow three three steps:

1. Step: Create An Enumeration

2. Step: Add Values

3. Step: Select Enumeration As The Data Type


1. Step: Create An Enumeration

Click on 'Model' in the top menu, select 'Enumeration' and then 'Create'. A window will open where you can enter the name of your enumeration (e.g., "RatingEnum").

Note: In this step, you are not yet adding values to your enumeration.


2. Step: Add Values

Click on 'Model' in the top menu again, select 'Enumeration' and then 'Edit'. Select an enumeration from the 'Choose Enumeration' drop-down menu. Then, click on 'Add Element' at the bottom of the window, enter the value (e.g. "G") in the new window and close the window.

Repeat this process for all other values and close the window when finished.

Note: The third step will be explained in the next task, as it is not yet necessary for this subtask.
