2. Theoretical Background:

Instantiation refers to the process of creating an object from a given class. In class-based object-oriented languages (such as the UML, Java, and most other object-oriented languages), the specification of a class must always precede the creation of objects. Each class has a unique name and specifies different properties. Properties of classes are attributes, associations, operations, and constraints. In this learning unit, we focus on attributes only. Associations are discussed in LU2, operations in LU9, and constraints in LU10. Classes may contain an arbitrary amount of attributes. Each attribute consists of a name and a type. The type of an attribute determines its range of allowed values. Some data types are included as part of UML++. These include: String (for any textual input), integer (for integers), float (for floating-point numbers), and Boolean (for Boolean values). More advanced, custom types for attributes are discussed in LU4. The term initialization refers to the initial assignment of attribute values to objects. Initialization can only succeed the instantiation of an object. The value of an attribute in a given object is called the slot of an object.
